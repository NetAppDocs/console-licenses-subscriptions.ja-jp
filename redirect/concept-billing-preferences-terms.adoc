= 用語と定義
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
課金設定は、 NetApp Consoleがライセンスとサブスクリプションの支払い、請求書、更新動作を処理する方法を制御します。設定は 2 つのレベルのいずれかで行います。

* *グローバル* — 直接ライセンス、マーケットプレイス サブスクリプション、 Keystoneサブスクリプションに適用されます。
* *サブスクリプションごと* — 特定のマーケットプレイスまたは直接ライセンス契約にのみ適用されます。


請求元は次のとおりです：

* *マーケットプレイス サブスクリプション* — 課金は AWS Marketplace、Azure Marketplace、または Google Cloud Marketplace を通じて行われます。クラウド マーケットプレイスが提供する請求アカウント、リージョン、更新オプションを選択できます。
* *直接ライセンス* — 課金はNetAppを通じて行われます。ライセンス キー、請求書の連絡先、自動更新はNetApp Consoleで管理します。
* * Keystoneサブスクリプション* — 課金はKeystone契約を通じて行われます。請求書の受取人とコスト センター タグは、NetApp Consoleで管理します。




== 権限

請求設定を変更するには、*アカウント管理者*または*請求管理者*の役割が必要です。オペレーターは設定を閲覧することはできますが、変更することはできません。



== 請求設定を行う

課金設定により、次のことが可能になります。- サービス全体で単一の請求書連絡先リストを適用する、- 使用状況をコスト センターに合わせる、- 自動更新と有効期限の通知を管理する、- 新しいCloud Volumes ONTAP (CVO) システムが正しい課金アカウントを継承するようにする。



= 請求フィールド



== マーケットプレイスフィールド

* *マーケットプレイス プロバイダ* — AWS、Azure、または Google Cloud。
* *請求アカウント / サブスクリプション ID* — 課金対象となるクラウド請求アカウント。
* *契約期間* — 契約期間の長さと更新動作。




= 請求オプションと定義



== 条件とオプション

* **NetApp ライセンス優先** — アクティブな NetApp ライセンスに対して請求されます。追加使用分はマーケットプレイスに移行します。
* **マーケットプレイスのサブスクリプションのみ** — すべての使用量は、マーケットプレイスの契約を通じて請求されます。
* **PAYGO（Pay-As-You-Go）** — 実際の使用量に応じて課金される柔軟なサブスクリプション。
* **年間契約** — 固定期間の Marketplace サブスクリプション。
* **自動更新** — 契約満了時に自動的に契約が更新されます。
* **カスタムCVO構成** — クラウド プロバイダごとに複数のサブスクリプションをマッピングするための高度なモード。
* **直接ライセンス** — マーケットプレイス容量の前に使用されるNetApp管理ライセンス。
* **ハイパースケーラー** — AWS、Azure、またはGoogle Cloud プロバイダ。
* **変更内容の確認ダイアログ** — 意図しない請求内容の変更を防ぐため、アップデートを適用する前に表示されます。




== 一般的な構成パス

|===
| シナリオ | 推奨される請求オプション |  


| 購入したNetApp容量の使用 | NetApp ライセンスを最初に取得 |  


| マーケットプレイス契約のみを使用する | マーケットプレイスのサブスクリプションのみ |  


| 1つのクラウド プロバイダで複数のプロジェクトに請求 | カスタムCVO設定を有効にする |  


| デフォルトの請求方法に戻す | カスタムCVO設定を無効にする |  
|===
