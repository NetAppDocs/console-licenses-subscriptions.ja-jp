= 用語と定義 :hardbreaks: :nofooter: :icons: フォント :linkattrs: :imagesdir: ./media/

[role="lead"]
課金設定は、 NetApp Consoleがライセンスとサブスクリプションの支払い、請求書、更新動作を処理する方法を制御します。設定は 2 つのレベルのいずれかで行います。

* *グローバル* — 直接ライセンス、マーケットプレイス サブスクリプション、 Keystoneサブスクリプションに適用されます。
* *サブスクリプションごと* — 特定のマーケットプレイスまたは直接ライセンス契約にのみ適用されます。


請求元は次のようになります。

* *マーケットプレイス サブスクリプション* — 課金は AWS Marketplace、Azure Marketplace、または Google Cloud Marketplace を通じて行われます。クラウド マーケットプレイスが提供する請求アカウント、リージョン、更新オプションを選択できます。
* *直接ライセンス* — 課金はNetAppを通じて行われます。ライセンス キー、請求書の連絡先、自動更新はNetApp Consoleで管理します。
* * Keystoneサブスクリプション* — 課金はKeystone契約を通じて行われます。請求書の受取人とコスト センター タグは、NetApp Consoleで管理します。


== 権限 課金設定を変更するには、*アカウント管理者* または *課金管理者* の役割が必要です。オペレーターは設定を表示することはできますが、変更することはできません。

== 請求設定を行う

課金設定により、次のことが可能になります。- サービス全体で単一の請求書連絡先リストを適用する、- 使用状況をコスト センターに合わせる、- 自動更新と有効期限の通知を管理する、- 新しいCloud Volumes ONTAP (CVO) システムが正しい課金アカウントを継承するようにする。

= 請求フィールド :page-role: 参照

== マーケットプレイス フィールド * *マーケットプレイス プロバイダー* — AWS、Azure、または Google Cloud。  * *請求アカウント/サブスクリプション ID* — 課金対象のクラウド請求アカウント。  * *契約期間* — 期間の長さと更新の動作。

= 請求オプションと定義 :page-role: リファレンス

== 条件とオプション * ** NetAppライセンスが優先** — アクティブなNetAppライセンスに対して請求されます。追加の用途はマーケットプレイスに移行します。  * **マーケットプレイス サブスクリプションのみ** — すべての使用量はマーケットプレイス契約を通じて請求されます。  * **PAYGO (Pay-As-You-Go)** — 実際の使用量に応じて課金される柔軟なサブスクリプション。  * **年間契約** — 固定期間のマーケットプレイス サブスクリプション。  * **自動更新** — 契約期限が切れると自動的に契約を更新します。  * **カスタム CVO 構成** — ハイパースケーラーごとに複数のサブスクリプションをマッピングするための高度なモード。  * **直接ライセンス** — マーケットプレイスの容量の前に使用されるNetApp管理ライセンス。  * **ハイパースケーラー** — AWS、Azure、または Google Cloud プロバイダー。  * **変更の確認ダイアログ** — 誤って請求が変更されるのを防ぐために、更新を適用する前に表示されます。

== 一般的な構成パス | シナリオ | 推奨される課金オプション | |----------|-----------------------------| | 購入したNetApp容量の使用 | NetAppライセンスを優先 | | マーケットプレイス契約のみの使用 | マーケットプレイス サブスクリプションのみ | | 1 つのクラウド プロバイダーの下で複数のプロジェクトに課金 | カスタム CVO 構成を有効にする | | デフォルトの課金に戻す | カスタム CVO 構成を無効にする |
